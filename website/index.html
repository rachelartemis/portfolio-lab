TLS 1.3 Handshake Analysis

Captured live on the loopback interface during Lab 09. Every packet below was observed in Wireshark when the browser first connected to https://localhost.

01
Client Hello
TLSv1.3 · 1879 bytes · SNI=localhost

Browser sends supported TLS versions, cipher suites, and a random nonce. Fully readable in Wireshark — not yet encrypted.

02
Server Hello + Certificate
TLSv1.3 · 3276 bytes · Server Hello, Change Cipher Spec

Nginx replies with selected cipher suite and sends nginx.crt (RSA-4096). TLS 1.3 bundles all in one flight.

03
Key Exchange — DHE
Diffie-Hellman Ephemeral · Forward Secrecy

Both sides independently compute the same session key. The shared secret never travels over the wire — forward secrecy.

04
Change Cipher Spec
TLSv1.3 · 94 bytes · Both sides confirm

Both sides signal that all subsequent packets will use the negotiated symmetric session key.

05
Encrypted Application Data
TLSv1.3 · Application Data · HTTPS Active

All HTTP content is encrypted. Wireshark shows only opaque Application Data — nothing readable. HTTPS works.
