TLS 1.3 Handshake — Live Analysis

Each stage was captured in Wireshark during this lab. The sequence below shows exactly what happened when the browser connected to https://localhost.

01
Client Hello
TLSv1.3 · Client Hello (SNI=localhost)
Browser sends supported TLS versions, cipher suites, and SNI=localhost. Not yet encrypted — visible in plain text in Wireshark.
↓
02
Server Hello + Certificate
TLSv1.3 · Server Hello, Change Cipher Spec, Application Data
Nginx selects TLSv1.3 and cipher suite. Sends nginx.crt with public key. TLS 1.3 bundles all of this in one flight.
↓
03
Key Exchange (DHE)
TLSv1.3 · Change Cipher Spec, Application Data
Both sides independently compute the same session key via Diffie-Hellman Ephemeral. The shared secret never travels over the network.
↓
04
Change Cipher Spec
TLSv1.3 · Change Cipher Spec (94 bytes)
Both sides signal that all subsequent packets will be encrypted using the negotiated session key.
↓
05
Encrypted Application Data
TLSv1.3 · Application Data (all subsequent packets)
All HTTP content is now encrypted. Wireshark shows only opaque Application Data — no HTML, no headers, nothing readable. HTTPS is working.
